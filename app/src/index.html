What are Walrus Sites?

DEFINITION

Fully decentralized, cost-efficient, and globally available websites powered by Walrus and Sui

Traditional website hosting

  • 01. Hosting Provider

    Traditional website hosting begins with selecting a hosting provider that offers server space to store your website's files.

  • 02. File Upload

    Website files, including HTML, CSS, and media, are uploaded to the hosting provider's servers, typically via FTP or a control panel.

  • 03. Domain Connection

    A domain name is purchased and configured to point to the hosting provider's servers, enabling users to access the website via the internet.

  • 04. Maintenance

    • Server Monitoring
    • Software Updates
    • Backup Management
    • Performance Optimization

Walrus Sites

  • 01. Hosting Provider

    Using a hosting provider for your site is not needed anymore. Sites can be accessed through self-hosted or third-party portals.

  • 02. File Upload = Walrus + Sui

    Source files are stored on Walrus as blobs. A Sui smart contract is used to keep track of the mapping between blobs and site resources.

  • 03. [Optional] Domain Connection: SuiNS

    Optionally, your site can be accessed through other portals using SuiNS.

  • 04. Maintenance

    • Portal Monitoring
    • Software Updates (Move Calls)
    • Backup Management

Architecture Overview

An overview of the components that make up Walrus Sites.

Sui in one slide

* * *

A Layer-1 blockchain, known for its object-centric data model and the Move programming language.

In Sui you can write smart contracts. Imagine Sui being a big decentralised database, and the contracts being complex queries.

You can write smart contracts in the Move language.

Walrus in one slide

* * *

It enables builders to store, retrieve, manage, and program large data and media files, such as videos, images, and PDFs.

Files can be stored as blobs, making them accessible to anyone by using a blob id.

While Walrus alone has limited functionality, its true potential is unlocked when integrated with platforms like Sui (or any other chain).

The site-builder CLI

An overview of the site-builder CLI.

This is the tool that provides an interface (write/update) to interact with your walrus site.

# Deploy a site
site-builder --context=<testnet|mainnet> deploy <build-directory>

# Show the pages composing the site
site-builder --context=<testnet|mainnet> sitemap <object_id | @suinsname>

# Update a site
site-builder --context=<testnet|mainnet> update <object_id> <build-directory>

FAQs

Walrus Sites is an open source solution. If you have additional questions, join our discord server!

Why Walrus Sites?

Decentralize the stack. Assets (Sui) -> Data (Walrus) -> Distribution (Walrus Sites). End-to-end verifiability. Censorship-resistant.

* * *

What is the cost of a Walrus Site?

Depends on the storage duration (epoch) and blob size. As a rule of thumb: 1GB for 1 month ‚âà 0.2 WAL.

When does a Walrus Site expire?

A site expires when at least one blob of a crucial resource (neuralgic point) expires. You can always update your site and extend its' lifetime.

* * *

Can I host a backend on Walrus Sites?

No because resources stored on Walrus are static. Walrus Sites act like frontend servers. You can also replace your backend with a smart contract for complete decentralization.
